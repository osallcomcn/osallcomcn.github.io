**关键路径** vs **关键链**

| 关键路径                                                     | 关键链                                                       |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 关键路径法是在进度模型中，估算项目最短工期，确定逻辑网络路径的进度灵活性大小的一种方法。 | 关键链发（CCM）是一种进度规划方法，允许项目团队在任何项目进度路径上设置缓冲，以应对资源限制和项目不确定性。<br />**资源约束性关键路径就是关键链。**<br />项目缓冲放置在关键链末端的缓冲，用来保证项目不因关键链的延误而延误；<br />接驳缓冲则放置在非关键链与关键链的结合点，用来保护关键链不受非关键链延误的影响。 |
| 理想化的进度规划方法                                         | 考虑了资源约束和不确定性                                     |

关键链

- 问题提出<u>***帕金森定律***</u>：工作会自动膨胀占满所有可用时间。如果安排给一个任务的时间富余，人们就会放慢节奏消耗掉所有富余时间。
- 解决方案：最早开始法则 所有活动都越早越好，砍掉每个活动的安全时间，集中到路径末端就是准备项目缓冲。



**偏差分析** vs **趋势分析**

| 工具/技术 | 定义                                                         |
| --------- | ------------------------------------------------------------ |
| 偏差分析  | 偏差分析审查<u>**目标绩效和实际绩效之间**</u>的差异（或偏差），可设计持续时间估算、成本估算、资源使用、资源费率、技术绩效和其他测量指标。 |
| 趋势分析  | 趋势分析<u>**根据以往结果预测未来绩效**</u>，它可以预测项目的进度延误，提前让项目经理意识到，<u>按照既定趋势发展，后期进展可能出现的问题</u>。应该在足够早的项目时间进行趋势分析，使项目团队又时间分析和纠正任何异常。可以根据趋势分析的结果，提出必要的预防措施建议。 |



|          | 偏差分析                                                     | 趋势分析                                                     |
| -------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 联系     | 都是数据分析工具的分支；<br />多用于监控过程组，对数据、信息、绩效进行处理的工具/技术；<br />都可能产生变更请求。 |                                                              |
| 区别     | 基于基准，找到<u>差距</u>；<br />是进行趋势分析的前提；<br />变更请求的类型多为<u>纠正或补救</u>。 | 基于当前，<u>预测未来的走势</u>；<br />可以成为偏差分析的基线；<br />变更请求的类型多为<u>预防</u>。 |
| 王安解读 | **已经发生，站在现在看现在**                                 | **尚未发生，站在现在看未来**                                 |



## 冲突管理

### 冲突来源

| 进度计划               | 项目任务的进度安排、任务的排序和时间选择存在不同意见         |
| ---------------------- | ------------------------------------------------------------ |
| 项目优先级             | 相关方在任务的优先顺序上有不同意见（项目前期）               |
| 资源稀缺               | 资源匮乏（项目前期）                                         |
| 技术意见与执行情况权衡 | 在技术问题与执行情况权衡上，存在不一致意见                   |
| 行政程序上的冲突       | 在如何管理项目的问题上，发生的项目管理和行政管理程序之间的冲突 |
| 成本                   | 在涉及到工作分解结构上，来自支持部门的成本估算上的冲突       |
| 个人工作风格/个性      | 人际关系方面的冲突（项目中期）                               |





王安解读：八字方针：”直接”、“合作”、“尽早”、“私下”管理冲突。
